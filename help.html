The composer

One of the main reasons of its existence is front and center: a large, blank workspace for writing and composing texts.

In contrast to applications offering limited space, often confined to the size of an SMS box, this utility prioritizes ample room for writing informative, high-quality content.

Document formatting in Markdown

While the purest of pure is simply plain text, you can also write in Markdown which is a lightweight markup language that allows you to format text using simple symbols. You can easily add headers, bold or italicize text, create lists, and insert links—all using Markdown's straightforward syntax.

Document interactivity with a Mark

In addition to writing Markdown syntax, you can also add a Mark (aka Tweak, Twist, Curly) which is a pair of double curly brackets {{}} that become automatically highlighted and interactive with keyboard and mouse. Think of it as an easy way to create a form inside your writing that you can interact with.

Mark a position in the text that requires attention:

Get: {{}}

Provide a hint by wrapping the text in comment syntax /**/:

Get: {{/* shopping items */}}

Provide a default option when selected:

Get: {{ water }}

Provide a list of options by separating them with the pipe | character:

Get: {{ water | milk | beer | wine }}

Text inside a Mark is automatically trimmed, meaning whitespace before and after is removed. This gives the text some space to breathe and make it more readable.

Put a long list of options under each other:

Get: {{
    water |
    milk |
    beer |
    wine |
    bread |
    spaghetti
}}

And maybe the option pipe separator on the left:

Get: {{
        water
    |   milk
    |   beer
    |   wine
    |   bread
    |   spaghetti
}}

If you don't want the text to be trimmed, use an exclamation mark ! right after the opening curly brackets:

Try it out.{{! This will keep the space...
and the line break.}}

Document auto-save and load

Documents are automatically saved, preventing any loss of work. When you reopen the application, it loads the most recent document you were working on, so you can pick up right where you left off without any hassle.

Important: documents that haven't been modified in 14 days will automatically be deleted.

The content library

Aside the large textarea is a place to create, organise and select snippets of content that you find important and useful. A snippet can be a piece text you want to reuse, a knowledge document you want to consult, or a script you come up to automate certain parts of your writing.

The content library allows you to manage two types of content:

  • Snippet (aka static snippet)
  • Macro (aka scripted snippet)

Newly created snippets and macros automatically land in the inbox. You can create folders to organise your content with drag-and-drop and collapse/expand them for easier navigation once the library becomes large.

Snippet

A snippet is a piece of text which allows you to quickly insert frequently used phrases, templates, or boilerplate text, reduce repetitive typing and save time.

A snippet is defined by:

  • title: a short description, used to explain the concept of the snippet
  • trigger: a unique, case-insensitive word used to identify the snippet and "trigger" it while typing or referencing it in other snippets
  • note: optional content in plain or markdown-formatted text to provide extra information about the snippet
  • text: the actual snippet, written in plain or markdown-formatted text with Mark and can be injected into your writing by selecting it from the library. A snippet has access to other snippets and macros, allowing you to compose an entire document automatically from different individual snippets of text.

A snippet can have multiple texts, published or drafted, and reordered with drag-and-drop.

Macro

A macro is a piece of text written in JavaScript to create instructions which use and transform data. It cannot reference and use other snippets or macros, but it is context-aware and has extra capabilities giving it some powers to automate your work and environment.

A macro - just like a static snippet - is defined by:

  • title: a short description, used to explain the concept of the macro
  • trigger: a unique, case-insensitive word, used to identify the macro and "trigger" it while typing or referencing it in other snippets
  • note: optional content in plain or markdown-formatted text to provide extra information about the macro
  • script: the actual instructions written in JavaScript which are automatically wrapped in a function. The returned value can be used in other snippets or injected directly into your document.

A macro can have multiple scripts, published or drafted, and reordered with drag-and-drop.

Content examples

Imagine a commonly written text:

Hi {{/* human */}},

Thank you for getting in touch.

Kind regards,
Another human

Break it down into multiple, reusable snippets:

  • $hai
    Hi {{/* human */}},
  • $ty
    Thank you for getting in touch.
  • $sign
    Kind regards,
Another human

Now you can individually "trigger" those snippets by:

  • selecting them in the content library, or
  • writing their trigger words (without the $ prefix) and hit Tab to expand the snippet
    hai Tab
    ty Tab
    sign Tab

With just 3 clicks, or 3 word x Tab hits, you can compose an entire text as fast as you can click or write.

In case this is a common block of text and you prefer a single click experience, combine the individual snippets into a canned response... even with a {{Mark}}:

  • $cr
    $hai

$ty

{{/* write here */}}

$sign

While this is a contrived and minimal example, the ability to compose a document or template by combining various snippets of text with just a reference becomes very powerful. And keep in mind that snippets can also use macros...

  • $DayOfTheWeek Get today's weekday
    // Get today's date as a string in the format 'weekday'
return new Date().toLocaleDateString('en', {weekday: 'long'});
    ... and use it in the $ty snippet
    Thank you for getting in touch on this lovely $DayOfTheWeek.
  • $DomainOfContact Get the domain name from the mail sender
    // Get the email address from the mail sender
const emailAddress = $mail.sender.email;
// And return the part after the @
return `${emailAddress}`.split("@")[1];
    ... and use it in a snippet
    I checked out $DomainOfContact and found it... interesting.
  • $Watman obligatory example to show off JavaScript´s powers
    return Array(16).join("wat" - 1) + " Batman!";

Macro document context and directives

Inside a macro, you also:

  • have access to the $hai document context, i.e. the document you are writing, and
  • can modify the same $hai document context by using directives (i.e. JavaScript comments in a predefined format)

This means you can use text you wrote, run it through your own program, and rewrite the text programmatically.

There are two types of directives available. The first one directs when the macro is run, i.e. on:load or on:trigger.

  • Direct the macro to run when the content and/or context is loaded:

    // hai on:load
return "I'm here";

    The script will be executed automatically and make the returned value available to be used in snippets. This is the default behaviour of a macro which means defining on:load is not really necessary.

  • Direct the macro to run when triggered:

    // hai on:trigger
return "I'm here because you triggered me";

    The script will be executed when you invoke it, which is especially helpful when working with dates and times that need to be correct up-to-the-minute or second. It also allows you to run a script that works with the text you are writing.
    Note: macros that run on:trigger cannot be used in snippets

The second one directs what the returned value will be used for, i.e. set:<document section>.

  • Direct the macro to use the returned value as the document content on load:

    // hai set:docContent
return "Now I'm the content";

Warning: directives are a powerful way to automate and prepare your compositions with combinations of when and what to write. You can in fact automatically compose a document on:load, but be careful to not overwrite whatever you have written before.

  • $LoudContent to transform the writing in all CAPS:

    // hai on:trigger
// hai set:docContent
// First get the content
const content = $hai.docContent;
// Then transform it
const loudContent = content.toUpperCase();
// Now return the transformed content and overwrite
// the current content as directed by set:docContent 
return loudContent;

  • When finished writing, trigger $BuildFootnotes turn markdown-style inline links into footnotes

    // hai on:trigger
// hai set:docContent
// Get the written text
let text = $hai.docContent;
// Regular expression to match markdown-style links
const mdLinkRE = /\[([^\]]+)\]\(([^)]+)\)/gm;
// Array to store footnotes
let footnotes = [];
// Index for footnotes (start at 1)
let idx = 1;
// Find all matches of markdown-style links
const matches = [...text.matchAll(mdLinkRE)];
// Iterate over each match
matches.forEach(([match, title, link]) => {
  // Add footnote to the footnotes array
  footnotes.push(`[${idx}] ${link}`);
  // Replace the markdown-style link with the title and footnote index
  text = text.replace(match, `${title} [${idx}]`);
  // Increment the footnote index
  idx++;
});
// If no footnotes were generated, return the original text
if (!footnotes.length) return text;
// Determine whitespace based on whether there are newlines at the end of the text
const whitespace = /\n\s*\n\s*$/.test(text) ? '' : '\r\n\r\n';
// Concatenate the original text, whitespace (if needed), and footnotes, and return
return text + whitespace + footnotes.join('\r\n');

Document header customisation

There are two more set directives available:

  • // hai set:docHeaders1
  • // hai set:docHeaders2

A macro can use these directives to programmatically set the document headers.
Both the main header (docHeaders1) and sub header (docHeaders2) get two lines after which the text will be truncated. This means there is room for up to 4 lines of header text that can be optionally styled with HTML tags: h1, h2, h3, h4, strong, em, and hr.

  • Set the main header when loading a document
    // hai set:docHeaders1
return `<h1>My notes</h1>`;
  • Set the sub header with today's date when loading a document
    // hai set:docHeaders2
// Get the current date as a string representing the date only
const today = new Date(Date.now()).toDateString();
// Return the current date with some emphasis
return `<em>${today}</em>\n<hr>`;
  • When finished writing, add some info to see how long it will take for someone to read it
    // hai on:trigger
// hai set:docHeaders2
// Get the written text
let text = $hai.docContent;
// Trim the text and split it into words
const words = text.trim().split(/\s+/).length;
// Average reading speed in words per minute (wpm)
const averageReadingSpeed = 183;
// Calculate the estimated reading time in minutes (rounded up)
const estimatedMinutes = Math.ceil(words / averageReadingSpeed);
// Generate the message indicating the estimated reading time
const readingTimeMessage = `(About a ${estimatedMinutes} minute read)`;
// Return the reading time message
return readingTimeMessage;

The context manager

Context (from Latin contextus, con- 'together' + texere 'to weave') is the source of information you use when you are composing notes, email replies, knowledge documents, etc. It is the webpage you visit or have open all the time in your browser to get something done.

If you need to reply to a lot of emails, Gmail or Outlook is the context of your writing. If you interact with customers, Zendesk or Jira might be the resources you use all the time. If you do research, maybe Wikipedia is your source of information. If you are a developer, GitHub issues or discussions might be the context you write and respond to a lot.

Context is not required to run the application, but when you do, it becomes much more useful because your snippets and macros - and therefore your writing - becomes instantly context-aware.

Context examples

Imagine you provided a mail context with the following structure:

mail 
├── subject
├── from 
│   ├── name 
│   └── email 
└── received

Back in your webmail application, if you want to reply to an email, click the browser extension icon and Hai will source the context according to your instructions and make it available in a tree of information:

mail 
├── subject: Ideas for improvement 
├── from 
│   ├── name: Jane Doe 
│   └── email: jane.doe@acme.com 
└── received: 2 hours ago

Just as in the snippet library, you can click the individual items to insert them in the text, or reference it inside your snippets and macros with the $ symbol, followed by its name and the "path" to the information you want to inject.

  • $hai snippet with the contact name automatically injected
    Hi $mail.from.name,
  • $SenderFirstName, because the full name is a bit formal
    const fullName = $mail.from.name;
// Split the full name by spaces and take the first part
return fullName.split(" ")[0];
    ... now we can salute the sender in $hai by their first name:
    Hi $SenderFirstName,
    Or maybe give the option of which part to pick with a Mark
    const fullName = $mail.from.name;
// Split the full name into parts and join them with a pipe
const nameParts = fullName.split(" ").join(" | ");
// and return it wrapped in a textmark as a selector
return `{{ ${nameParts} }}`;
  • $wrt A snippet of text using the subject
    With regards to the topic "$mail.subject".
  • $MailSubject header so we know what the subject is about
    // hai set:docHeaders1
return $mail.subject;
  • $MailInfo header with some extra details
    // hai set:docHeaders2
// example: Jane Doe (2 hours ago)
const mailFrom = $mail.from.name;
const mailReceived = $mail.received;
return `${mailFrom} (${mailReceived})`;
  • $AutoSpamReply to not waste too much time
    // hai set:docContent
// Get the subject of the email
const subject = $mail.subject;

if (subject.includes("win major prizes")) {
  return `Hi,

Thank you for reaching out.

At the moment I have no interest in winning major prizes.
However, please do stay in touch in case I won.

Kind regards,
Lucky Luke`;
}
